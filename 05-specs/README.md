## Current rule

Do not build from a specification merely because it exists.

Before implementation, confirm:

- governing decisions are accepted;
- unresolved research dependencies are addressed or explicitly deferred;
- open correctness questions are closed;
- validation and hammer-test requirements are defined;
- the specification has passed the required audit and human review;
- a governed task packet authorizes the implementation scope.
