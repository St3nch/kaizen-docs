```text

It should be usable across multiple SERP / SEO / LLM citation / marketplace intelligence projects.
```


This question may produce a decision candidate later, but 020B does not decide it.

## 9. Reserved-vs-built repo rule

During repo reconciliation, Kaizen must distinguish:

```text
built and tested code;
reserved folder shape;
docs-only planning;
mock/sample artifacts;
private/git-ignored local artifacts;
accepted doctrine;
draft or historical docs.
```

A reserved folder or empty app shape is not proof of a built feature.

A docs-only project must not be described as a built software system.

## 10. Private / git-ignored data rule

M14 must not request, infer, or fabricate private or git-ignored data.

If repo docs reference private local artifacts, record:

Do not invent contents.

## 13. Reconciliation classifications

All generated-doc claims compared to repo reality must be classified as one of:

```text
CONFIRMED — generated claim matches repo reality.
CONTRADICTED — repo reality differs.
STALE — claim or repo has moved past the other.
INVENTED — generated claim has no support.
RESERVED-NOT-BUILT — generated docs treated reserved/deferred structure as built.
PRIVATE/LOCAL — data exists or is referenced but is intentionally local/private/out of scope.
UNKNOWN — insufficient evidence; do not infer.
```
