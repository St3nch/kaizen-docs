# Kaizen Docs Repo

This repository is the planning and stewardship workspace for the Kaizen project.

Kaizen is a project intelligence pipeline designed to move work from rough idea capture to research, synthesis, spec planning, audit, and implementation-ready handoff packets for coding agents.

This repo is **not** the future Obsidian vault. It is where we design the vault standard, collect research, define rules, and prepare implementation documents before anything becomes operational doctrine.

## Start here

LLMs and humans should begin with:

- [`00-entrypoint/LLM_START_HERE.md`](00-entrypoint/LLM_START_HERE.md)

That file explains the folder map, traversal order, context, rules, and current project posture.

## Current status

- Project stage: early planning / research setup
- Source of truth: this repo for Kaizen planning docs only
- Future Obsidian vault: not created yet
- Hermes write access: not approved yet

## Stewardship rule

Do not turn research findings directly into doctrine. Research must be promoted through review:

```text
raw research -> summary -> claims/constraints -> proposed standard change -> review -> accepted doctrine
```

## Repository purpose

This repo holds:

- Kaizen project standard drafts
- deep research prompts
- research result summaries
- design decisions
- spec drafts
- Hermes capability and boundary docs
- Obsidian architecture notes
- handoff packet patterns

This repo does **not** hold:

- running code
- the actual Obsidian vault
- private customer data
- generated build artifacts
- active business operations
